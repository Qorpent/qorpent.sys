= Форматирование слов и символов
Поддерживаются следующие символы форматирования участков текста:
[[code]]
Выделение участка текста ***жирным***
Выделение участка текста **курсивом**
Выделение участка текста __подчеркиванием__
Перечеркивание участка текста --вот такой чертой--
Выделение участка текста подстрочником, например X,,i,,
Или наоборот надстрочником, например x::2::
Также можно выделить {style:color:red;background-color:blue;font-size:14pt;font-family:Tahoma}любым свободным стилем CSS{style}
Можно свободно комбинировать: *****::--__тут почти вся сборная солянка форматирования__--::*****
[[/code]]
При выполннии кода сверху результат:
[[sample]]
% Выделение участка текста ***жирным***
% Выделение участка текста **курсивом**
% Выделение участка текста __подчеркиванием__
% Перечеркивание участка текста --подчеркиванием--
% Выделение участка текста подстрочником, например X,,i,,
% Или наоборот надстрочником, например x::2::
% Также можно выделить {style:color:red;background-color:blue;font-size:14pt;font-family:Tahoma}любым свободным стилем CSS{style}
% Можно свободно комбинировать: *****::--__тут почти вся сборная солянка форматирования__--::*****
[[/sample]]

{style:color:margenta;}Внимание!{style}
Все команды форматирования участков текста распространяются только в рамках одного абзаца (также внимательно прочитайте раздел про переносы строк)
----
= Заголовки
Заголовки указываются при помощи знаков = (равно), размещенных с самого начала строки, которая является заголовком.

Поддерживаются загловки до 6-го уровня включительно:

Например код:

[[code]]
= Первый уровень
== Второй уровень
=== Третий уровень
==== Четвертый уровень
===== Пятый уровень
====== Шестой уровень
[[/code]]

Будет отрисован как:
[[sample]]
= Первый уровень
== Второй уровень
=== Третий уровень
==== Четвертый уровень
===== Пятый уровень
====== Шестой уровень
[[/sample]]
----
= Списки
На данный момент используются только ненумерованные списки с точками (как более частые и понятные при форматировании)

Списки формируются подобно заголовкам, только в качестве первых символов строки используется знак % (процент)

Например код:

[[code]]
% Первый уровень
%% Второй уровень
%%% Третий уровень
%%%% Четвертый уровень
%%%%% Пятый уровень
%%%%%% Шестой уровень
[[/code]]

Будет отрисован как:
[[sample]]
% Первый уровень
%% Второй уровень
%%% Третий уровень
%%%% Четвертый уровень
%%%%% Пятый уровень
%%%%%% Шестой уровень
[[/sample]]

= Таблицы
% Таблицы состоят из идущих друг за другом строк
% Ячейки разделяются символом вертикальной черты
% Первая строка рассматривается как заголовок
% В ячейках можно использовать форматирование
% В специальном теге ячейки можно указать как объединять ячейки
% При использовании специального под-тега можно формировать дополнительные уровни заголовка
Рассмотрим пример:
[[code]]
|{,2}Название|{2}2010|{2}2011|{2}2012|{2}2013| 				 -- здесь есть теги {X,Y} указывающие число объединяемых колонок и строк, по умолчанию 1,1
|{+}кол|сумма|кол|сумма|кол|сумма|кол|сумма|    			-- здесь есть спецтег первой ячейки {+} которая указывает, что нужна еще одна строка заголовка
|--Пирожки--|{4}нет данных|5|100|{2}сняты с производства| -- тут есть объединения и форматирование
|***Конфеты***|1|10|2|20|10|100|100|1000|
[[/code]]
[[sample]]
|{,2}Название|{2}2010|{2}2011|{2}2012|{2}2013| 			
|{+}кол|сумма|кол|сумма|кол|сумма|кол|сумма|    		
|--Пирожки--|{4}нет данных|5|100|{2}сняты с производства|
|***Конфеты***|1|10|2|20|10|100|100|1000|
[[/sample]]

= Ссылки
Все ссылки формируются одинаковым синтаксисом
[[code]]
[http://yandex.ru] - пример ссылки, состоящей из одного адреса
[http://yandex.ru Яндекс!] - пример ссылки, состоящей из адреса и заголовка
[http://yandex.ru Яндекс! (на новой вкладке) | target='blank'] - пример ссылки, состоящей из адреса, заголовка и дополнительных атрибутов
[[/code]]
При отрисовке ссылки будут отрисованы так:
[[sample]]
% [http://yandex.ru] - пример ссылки, состоящей из одного адреса
% [http://yandex.ru Яндекс!] - пример ссылки, состоящей из адреса и заголовка
% [http://yandex.ru Яндекс! (на новой вкладке) | target='blank'] - пример ссылки, состоящей из адреса, заголовка и дополнительных атрибутов
[[/sample]]

Внимание! Способ отрисовки ссылок будет зависеть от настроек сайт и может отличаться от приведенного здесь, однако на способ описания ссылок это не влияет.

= Дополнительные блоки и специальные абзацы
Блоки кода формируются специальными блочными командами, синтаксис вызова которых приведен ниже. Для многих блоков существует открывающая и закрывающая часть, 
в этом случае за ними следует следить.
== Блок кода
Блоки кода выделяют зоны текста, использующие не форматирование вики, а форматирование исходного программного кода. 
Например на коде WIKI или C# или любого другого языка, так код:
[[code]]
__BLOCK__code]]
public int SQR (int y ) {
	return y * y;
}
__BLOCK__/code]]
[[/code]]

будет отрисован так:
[[code]]
public int SQR (int y ) {
	return y * y;
}
[[/code]]
(как включить внутрь блока кода описание другого блока кода смотрите в исходном тексте данной страницы)

== Блок примера
Является обычным блоком WIKI, но выделяется особым стилем, в данной справке все примеры выделены серым в рамке.
[[code]]
[[sample]]
Будет выделено особым стилем
[[/sample]]
[[/code]]

== Линия разрыва
Представляет собой строку из 4-х символов дефиса
[[code]]
----
[[/code]]
Будет отрисована так
[[sample]]
----
[[/sample]]
= Переносы строк
== Одинарный перенос внутри абзацев игнорируется
[[code]]
Это начало строки, потом просто перенос
и снова продолжили
[[/code]]
будет отрисован так:
[[sample]]
Это начало строки, потом просто перенос
и снова продолжили
[[/sample]]
Это позволяет писать большие абзацы без вставки ненужных в итоговом документе переносов, также поддерживается форматирование, с одиночным переносом

[[code]]
Это начало строки, ***потом просто перенос
и снова*** продолжили
[[/code]]

[[sample]]
Это начало строки, ***потом просто перенос
и снова*** продолжили
[[/sample]]

== Множественные переносы преобразуются в иденичный разрыв между абзацами
[[code]]
Это первый абзац

А это второй
[[/code]]
будет отрисован так:

[[sample]]
Это первый абзац

А это второй
[[/sample]]

== Дополнительно по переносам

Заголовки, списки и блоки кода не требуют дополнительного переноса

Так код:
[[code]]
= Заголовок
% Список
__BLOCK__code]] code __BLOCK__/code]]
[[/code]]
будет отрисован как 
[[sample]]
= Заголовок
% Список
[[code]] code [[/code]]
[[/sample]]

= Защита от форматирования WIKI
Если требуется напечатать участок текста с использованием символов WIKI, но так чтобы они не были преобразованы, то можно использовать 
защиту от преобразования

% на уровне отдельной строки можно использовать знак ! (восклицательный знак) в самом начале строки, при выводе страницы знак вопроса будет скрыт
например:

[[code]]
! здесь **текст** не будет __отформатирован__
[[/code]]

% выделить целый защищенный блок, например:
[[code]]
[[nowiki]]
здесь **текст** не будет __отформатирован__
и здесь --тоже--
[[/nowiki]]
[[/code]]
