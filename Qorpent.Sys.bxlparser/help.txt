[CYAN]
Call syntax:
[YELLOW]
bxlparser files [options,..,]
[DEFAULT]
0. files - any valid file path mask from current directory
u can use multiple using ";" as delimiter
[CYAN]
For example:
[YELLOW]
bxlparser my*.bxl;../base/*.bxl
[GREEN]
By default files will be converted into XML with UTF-8
encoding and places as "sourcefilename.bxl-parsed.xml"


[CYAN]
OPTIONS
All options are in 2 forms:
--OPTION
and
--OPTION VALUE
Available options:
[GREEN]
OUTPUT
[YELLOW]
--no-lex-info
[DEFAULT]
do not store lex info (source file, line) into result files
[YELLOW]
--output-format XML|JSON|BXL   (XML by default)
[DEFAULT]
Output format of convertation - XML - native XML file, JSON - file in JSON notation, BXL - if only 
include processing needed, not convertion
[YELLOW]
--output-directory DIRNAME
[DEFAULT]
Directory where result file will be created (in place side-by-side with sources by default)
[YELLOW]
--output-rename "REGEX;REPLACE"
[DEFAULT]
rename mask which will be applyed to source filename to setup result filename
[YELLOW]
--to-console
[DEFAULT]
renders result not to files, but directly in console
prefixize all files with format-based comment with source file name (<!-- SOURCE: filename --> , or /* SOURCE: filename */)

[GREEN]
INFORMATIONAL:
[YELLOW]
--help				
[DEFAULT]
shows this help
[YELLOW]
--enlist-sources
[DEFAULT]
shows list of source code files
